Day 1
Installation & Core Concepts
▼
What CPU Power Manager Does

A GNOME Shell Extension that runs inside the desktop itself, giving you a top-bar interface to control your CPU's driver, governor, and frequency limits through saved profiles. It writes directly to Linux kernel sysfs files.

Key Concepts
ConceptWhat It Is
intel_pstateIntel's CPU driver — talks directly to hardware
schedutilGovernor — sets scaling policy based on workload
Frequency ScalingCPU changes speed dynamically based on demand
GNOME ExtensionRuns INSIDE GNOME Shell — not a separate app
Installation Commands
bash
# Add PPA
sudo add-apt-repository ppa:fin1ger/cpupower

# Update package list
sudo apt update

# Install extension
sudo apt install gnome-shell-extension-cpupower

# Log out and back in, then enable
gnome-extensions enable cpupower@mko-sl.de
Key System Files — CPU Info
bash
# Current frequency (KHz) — divide by 1,000,000 for GHz
cat /sys/devices/system/cpu/cpu0/cpufreq/scaling_cur_freq

# Active driver
cat /sys/devices/system/cpu/cpu0/cpufreq/scaling_driver

# Active governor
cat /sys/devices/system/cpu/cpu0/cpufreq/scaling_governor

Polkit Security Rule

The extension uses a Polkit rule that embeds your username in the action ID. If requesting user = username in action → permission granted automatically. No password needed for profile switching.

location
/usr/share/polkit-1/rules.d/10-mko.cpupower.setcpufreq.rules
Known Issue — Preferences Window

Preferences window fails with Invalid object type 'GtkButtonBox' on GNOME 40+. GTK4 removed this widget. Known Bug Contribution Opportunity

Workaround: Use dconf directly from terminal.

Profile Storage Format

// Format: [MinFreq, MaxFreq, TurboBoost, Name, UUID]
[50, 100, false, 'ROS2 Mode', '111111111']
Write Profiles to dconf
bash
dconf write /org/gnome/shell/extensions/cpupower/profiles \
"[(50, 100, false, 'ROS2 Mode', '111111111'), \
(19, 60, false, 'Study Mode', '222222222'), \
(30, 80, false, 'Presentation Mode', '333333333')]"

# ⚠️ dconf OVERWRITES — always include ALL profiles
# TurboBoost must be true/false (lowercase)
Reload Extension After Changes
bash
gnome-extensions disable cpupower@mko-sl.de && sleep 2 && gnome-extensions enable cpupower@mko-sl.de
Live Frequency Monitoring
bash

# Watch current frequency across all cores
watch -n 1 "cpufreqctl --backend intel_pstate --format json info current"

intel_pstate Ground Truth Files
bash
# Read all intel_pstate settings at once
for f in /sys/devices/system/cpu/intel_pstate/*; do
  echo "=== $(basename $f) ==="; cat $f; echo
done

# Key files:
/sys/devices/system/cpu/intel_pstate/max_perf_pct  # max %
/sys/devices/system/cpu/intel_pstate/min_perf_pct  # min %
/sys/devices/system/cpu/intel_pstate/no_turbo      # 1=OFF 0=ON
/sys/devices/system/cpu/intel_pstate/num_pstates   # 35

⚠️ no_turbo=1 means Turbo OFF. no_turbo=0 means Turbo ON (double negative!)

Day 5
Automation & Scripts
▼
Auto Switch Configuration
bash
# Set battery profile (Study Mode)
dconf write /org/gnome/shell/extensions/cpupower/default-battery-profile "'222222222'"

# Set AC profile (ROS2 Mode)
dconf write /org/gnome/shell/extensions/cpupower/default-ac-profile "'111111111'"

# Enable Auto Switch
dconf write /org/gnome/shell/extensions/cpupower/auto-switch-profiles true
cpu-status Script
bash
#!/bin/bash
echo "======= CPU Power Status ======="
echo "Battery:     $(cat /sys/class/power_supply/BAT0/capacity)%"
echo "Status:      $(cat /sys/class/power_supply/BAT0/status)"
echo "AC Power:    $(cat /sys/class/power_supply/AC/online)"
echo "Max Freq:    $(cat /sys/devices/system/cpu/intel_pstate/max_perf_pct)%"
echo "Min Freq:    $(cat /sys/devices/system/cpu/intel_pstate/min_perf_pct)%"
echo "Turbo:       $(if [ $(cat /sys/devices/system/cpu/intel_pstate/no_turbo) -eq 1 ]; then echo 'OFF'; else echo 'ON'; fi)"
echo "Governor:    $(cat /sys/devices/system/cpu/cpu0/cpufreq/scaling_governor)"
FREQ=$(cat /sys/devices/system/cpu/cpu0/cpufreq/scaling_cur_freq)
echo "Current:     $(echo "scale=2; $FREQ/1000000" | bc) GHz"
echo "================================"
install
# Make executable and install system-wide
chmod +x ~/cpu-status.sh
sudo cp ~/cpu-status.sh /usr/local/bin/cpu-status

# Run from anywhere
cpu-status

cpufreqctl Direct Commands
bash
# Get current frequency info
cpufreqctl --backend intel_pstate --format json info current

# Get max frequency %
cpufreqctl --backend intel_pstate --format json max get

# Get min frequency %
cpufreqctl --backend intel_pstate --format json min get

# Get turbo state
cpufreqctl --backend intel_pstate --format json turbo get

# List available backends
cpufreqctl backends list
dconf Profile Management
bash
# Read all profiles
dconf read /org/gnome/shell/extensions/cpupower/profiles

# Read auto switch settings
dconf read /org/gnome/shell/extensions/cpupower/default-battery-profile
